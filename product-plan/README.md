## How to Use

### Option A: Incremental (Recommended)

Build your app milestone by milestone for better control:

1. Copy the `product-plan/` folder to your codebase
2. Start with Shell (`instructions/incremental/01-shell.md`)
3. For each section, open `prompts/section-prompt.md`, fill in the section variables, and copy/paste into your coding agent
4. Review and test after each milestone

### Option B: One-Shot

Build the entire app in one session:

1. Copy the `product-plan/` folder to your codebase
2. Open `prompts/one-shot-prompt.md`
3. Add any additional notes, then copy/paste into your coding agent
4. Answer the agent's clarifying questions and let it implement

## Tips

- **Always provide `product-overview.md`** alongside any instruction file — it gives essential product context
- **Use screenshots** in each section folder as visual references for fidelity checking
- **Read `tests.md`** before implementing each section — the test specs describe expected UI behavior in detail
- The components are props-based: they accept data and fire callbacks. Backend architecture, data fetching, and state management are your implementation decisions.
